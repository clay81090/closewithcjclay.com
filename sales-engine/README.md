# Sales Engine (3 phases)

Folders for HTSA’s prospect journey on **closewithcjclay.com**. Invoice workflow and skills stay separate from this tree.

| Phase | Folder | Role |
|--------|--------|------|
| **1 — Pre-call** | `sales-engine/pre-call/` | What prospects see before your call (low friction). *Work files land here as you split from `pre-call/` root if needed.* |
| **2 — Game plan** | `sales-engine/game-plan/` | On-call visual engine (pain → reality → vision → gap → vehicle → proof → game plan). |
| **3 — Enrollment** | `sales-engine/enrollment/` | Optional notes / links; **live invoices** remain root `htsa-enrollment-*.html` per existing workflow. |

**Version control:** Use Git commits on `main` (or feature branches) instead of many duplicate filenames. Tag milestones if you want: `git tag sales-engine-pre-call-v1`.

**Current state:** Codex snapshots live in `pre-call/snapshots/` and working copies in `pre-call/`. Reorganize into `sales-engine/*` when you move files—Git history keeps every step.
